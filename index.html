Leetcode 10. 正则表达式匹配

发表于 2017-10-25

这是一个简单的正则表达式匹配问题，解决方法主要为递归法和动态规划法。

题目

Implement regular expression matching with support for '.' and '*'.

'.' Matches any single character.
'*' Matches zero or more of the preceding element.

The matching should cover the entire input string (not partial).

The function prototype should be:
bool isMatch(const char *s, const char *p)

Some examples:
isMatch("aa","a") → false
isMatch("aa","aa") → true
isMatch("aaa","aa") → false
isMatch("aa", "a*") → true
isMatch("aa", ".*") → true
isMatch("ab", ".*") → true
isMatch("aab", "c*a*b") → true

'.' 代表任意字母， '*' 代表任意多个前字母。

方法一：递归法

该题只有 '*' 比较难处理，其他模式可以一个个单独进行比较即可。该讨论中省略对'.'的处理，方法无非就是如果遇到'.'返回true。

如果一个正则式出现 '*' ，意味着前面必然有一个非 '*' 的字符，因此在进行匹配时，我们需要对正则式两个两个地处理。
如果正则式分解到最后只有一个字符或者没有字符，那么我们直接比较即可，这就是递归结束的部分。

现在说递归的部分，也就是两个两个字符处理正则式。
先考虑较为简单的，假如两个字符都是非'*'字符，那么匹配的结果就是s[0] == p[0] && isMatch(s+1,p+1).为什么只比较第一个？因为第二个字符后面可能就是'*'，因此截取后面的字符串递归进行匹配。

然后考虑比较难的，如果第二个字符是'*'，那么匹配方式有两种。
一是'*'代表0个字符，那么匹配结果是isMatch(s,p+2)。'*'表示0个前字符说明前一个字符和'*'本身是没有意义的，直接从正则式中去掉也不影响匹配结果。
二是'*'代表一个或以上的字符，那么匹配结果是s[0]==p[0] && isMatch(s+1,p)。
